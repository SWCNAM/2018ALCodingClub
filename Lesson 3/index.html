Lesson 3: Learn about control flow!

Lesson for today:

Control flow are the words we use for code that helps the computer make decisions.

We are going to learn about a couple of specific kinds of control flow: if and switch statements.

If Statements

If statements tells the computer to do an action based on whether something is true or false.

For example: If I have at least 2 hours of free time then I will watch a movie. Or else, I will watch YouTube videos.

In JavaScript code, the 'if' statement for the example might look like this: 


    if(freeHours >= 2) {

        watchMovie();
    } else {

        watchYouTube();
    }
One interesting thing is that else part of the statement is actually optional! Sometimes you don't want to do anything if your deciding factor is not true. In that case, you can just write the if part and skip the else part.

For example, if it is raining, then I want to bring an umbrella with me. But if it's not raining, I don't want to do anything differently than I normally would.

What do you think that looks like in code? 

    if(isRaining) {            
        items.add(umbrella);
    }

Finally, if statements can get pretty complicated - you might see multiple ifs together, or even an if nested inside of another one! But as long as you figure out if each part is true or false, you'll be okay.
A complicated if statement might look like this: 

    if(isRaining) {
        // If it's raining, bring an umbrella!
        items.add(umbrella)

        if(temperatureOutside < 60){
            // If it's raining and also it's not warm outside, bring a jacket too.        
            items.add(rainJacket)
        }
    }

Switch Statements

Switch statements tell the computer to do actions based on multiple potential choices. Sometimes, just deciding between true or false doesn't give you all the options you want!

For example: Based on the current day of the week, I could either get ready for work in the morning, or sleep in.

In JavaScript code, that would look like this: 


    switch(today){
        case "Monday":
            getReady();
            break;
        case "Tuesday":
            getReady();
            break;
        case "Wednesday":
            getReady();
            break;
        case "Thursday":
            getReady();
            break
        case "Friday":
            getReady();
            break;
        case "Saturday":
            sleepIn();
            break;
        case "Sunday":
            sleepIn();
            break;          
    }
break is a special word that tells the computer to exit the switch statement. If we don't include that, then if today is Friday, you would end up doing the actions for Saturday and Sunday too (and accidentally sleeping in)!

You can use numbers inside of a switch statement, too. In fact, that's what we'll do for today's challenge!

Here's an example of what that could look like in our Brick Breaker game. Notice that you can skip numbers and that you can put case statements together if you want the same thing to happen for both of the numbers! 

    switch(brickBreakerLivesLeft){
        case 9:
        case 8:
            this.showMessage("You still have lots of lives left. Keep trying!");
        case 3:
            this.showMessage("Just a few lives left! Be careful!");
        case 1:
            this.showMessage("Uh oh! This is your last life!");                    
    }
